For this challenge I made a countdown with a targeted date.

The html layout

This is the html for the countdown. You can expect the same structure with the minutes and seconds. The real challenge is with javascript and a little bit of css.

The CSS styling

For flipping I mostly work with the ::before and ::after element. I do this because I don’t have to worry about changing the number using display but instead just show the half of the whole number. Styling the container, numbers and the segment is not a problem but the flipping animation will be.

First I will have to use overflow:hidden in order to not see the previous number inside the segment when flipping.

Next I will add the animations on the top-flip and bottom-flip. I will have to use position absolute on these two while position:relative is on the.flipcard. Now I am sure the animation will stay on the right place. After that I will basically flip the top 90 deg from the bottom, and the other way around with the bottom. After testing it out with a random number I saw that the animation worked well, so now I can start with the javascript.

The Javascript

Next I create a function that is responsible for displaying the remaining time until the new date I set. It also animate using the flip animation. First it calculates the numbers of seconds, minutes,hours and days remaining until the new date using ‘time’. Then it calls the flip() function to perform the flip animation for each second or unit passes. The document.querySelector("[data-hours-tens]" selects the flip and then with the math.floors it calculates the ten digits of number hours remaining. After that the flip() function is called. This is how It works for each unit.

After writing these two functions I set a setinterval() to execute the function repeatedly at a certain time. The const timeBetweenDates calculates the date between the current time and the date I wrote down. Next it will animate the right time using the flipAllCards function.

Next I want to have a flip animation. It first checks if the new number is the same as the current number on the flip card. If it is, the function returns without doing anything. If the new number is different it will create two new elements: topFlip and bottomFlip. This will be for the animation. I used pareInt() to compare the current number with the new number.

It adds an evenlisteners to these two to update the text content when the animation starts and ends, using the animationstarts and animationends. Finally it appends the two elements to the flip card to start the element.
