Calculator

Making a working calculator... Instead of using display:flex I thought of challenging myself with display:grid, since it also has its own perk.

Making the html page and css

The first thing I did was to make an container for the calculator. This time I wanted to use grid, just to test it out and to make the boxes uneven. Then I make a .output which displays the numbers you clicked. It will show the previous and current number. Next I made the buttons for the calculator, I used an iphone calculator as reference. I already wrote down the attributes so that I wont have to check it later when writing the code. The styling has nothing special going on aside from me using display:grid as a challenge.

Making the buttons inside Javascript

After that I will make the buttons functional using JavaScript. the first thing I thought of was how to display the data? First I will make a class calculator with a constructor which makes two arguments: previous and current. It will also use the .clear() method. This makes the calculator/input into an empty string. After making the class I will write down the functions I need to make the calculator functional: clear, delete, current and previous operand, the numbers and the operations and the equal sign. Now I will start making the functions functional. First lets I made a const to update the calculator input with the currentOperand and previousOperand. After that I will make an addeventlistener for every buttons. For each button it will use their corresponded function (for example the number buttons will use the appendnumber() and the operation button will use chooseoperation()) and then it will display it on the calculator using the updateDisplay(). I will use this method for each button on the calculator.

Next I will make the functions actually functional. The first thing I did was to make the input into a string. This is to make JavaScript actually append the string and not just add them to each other. For example if you click 1 + 1 it will become 2 and not 11. You can also see the if … return. This is to prevent the calculator to add multiple dots inside the string. So if the parameter (number) has an dot then it will return and stops the function.. functioning.

The next thing I made are the operations of the calculator. First I made the display using this.opreation.. etc. The empty string is to make the numbers which was clicked before the operation, go above. So the current operand is empty after using an operation. Next I want to prevent the to clear the display even if we have no current operand. To do this I used an if statement and it will check if the current operand is actually empty and if it does it will return which will end the function.

After making the two operation work it is time to make it actually calculate the numbers. To do this I will use the calculation function. For this I first have to make a variable for the result, this is the let calculation. Next is to make the previous and current typed string into an actual number using parseFloat(). Afterwards I will use the switch/case statement and define the operations. If there is none of the operations clicked then it will return.

Once this is done I will make the clear and delete function. The clear function is basically making the current and previous operand an empty string and the operation undefined. This way it will just erase the string and Javascript wont be using them. Then the clear function is making the current and previous operand into an string and then using the .slice() it will remove the string number/element once.
