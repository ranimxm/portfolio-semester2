The Code Implementation

Before dividing the pages, we had to make the basic elements (header, the block you see on the image below and the footer). We decided to do those things on my computer since I already had everything pulled from Git. Me, Melita and Iris were busy with the placements and fonts while Moussa and Thimo wrote down the colors for the root (Moussa greatly expressed how annoyed he was by sighing and just going to another group because we didn’t do it on his way but hey ).

I started out with the semester overview. I first did the first section of the page. This was easily done. Next was the columns, which was a bit harder to do. There were two problems for this part: how to hide the content and the placement of the image. The image placement was easily solved by using CSS transform and translate. The content also was easily solved thanks to the HTML5 element tag: details/summary. Due to the frontend assignment "classmates", I was able to solve this problem easily with no JavaScript.

At this moment I had two choices: I could just call it a day and strictly follow the figma design, or I could spice it a little bit up with animations. Of course I choose the second option.

The first element I animation was the arrow. If the details is not opened, it will show the left image. If it is opened, then it will show the left image. With some CSS styling, ::before and ::after tag I managed to animate it, however I want the animation to start when the details is toggled open.

To solve this I used a JavaScript function. for each time the DOM content is loaded, it will load the onclick function. With every click, it will start the animation on the class “collapsed”(which is the CSS animation), aka it will toggle.

The next animation is when the content of the detail(column) is displayed. Basically when the column is clicked on, you will see content. What I want to reach is when it is open, it will not show the content immediately but it goes from low to high opacity within a certain interval.

First I want the “.down”(the content) opacity go from 0 to 1. So I set the opacity to 0 on the start. After that the code will have to check if the details is opened or not, for this I used “isOpen= false”. Next I use an if/else statement. If the opacity is 0 then it will higher the opacity with an interval of 70 milliseconds (aka it goes from being closed to opened), and for the else: vice versa.

The last animation I added for this page is the icons and list inside the content. The code will check if the detail element is opened or not using the if/else statement. For each list it will add an animation using three classes. If it is not opened then it will remove the classes. This way the animation will keep repeating.

I finished this page faster than I thought + my part of the report already was done. After rechecking everything I noticed that a group member had a lot of tasks which it might be a lot to handle. I offered my help and made another page: the study overview.

The layout of this page was not a challenge at all since the other page had almost the same layout, but with more animations. The only “real” challenge on this page was the main section, see the image below. The line had to be longer than the width of the text. This was easily solved by using ::after and then margin-left -4em.

On both of the pages I made sure to make it responsive, which means it works on every device.

Perhaps you already noticed or maybe even not. I somehow became a big ‘fan’ using the if/else statements. In the future I have to try and find different ways of statements inside JavaScript.

The header

Since not everyone was willing to make the header or had something else to do, I volunteered myself to make the header. For this project, it consisted of a dropdown menu and a hamburger menu to make it responsive. On the image below you can see the html layout I used for the header. The hamburger icon is outside the header since it will effect the flex property I used on the .nav. Nex I put the dropdown list and inside of it I made a list of the dropdown items. Outside the items I put the pointer, in order not to get effected with the list underneath.

The next “hard” challenge was making the CSS. For the dropdown list I used keyframes and display:none. The keyframes are there to make a smooth animation (and to make the content displayed with using the visibility: hidden/visible). Using position absolute will make the item stay on its place and not go anywhere else.

Now for the pointer I had to use JavaScript. My plan was to add an class with JavaScript and to animate that certain class. This will start the keyframe whenever it will be clicked. But first I had to make the arrow. Firstly I will use the ::after/::before to style the up and bottom. Inside that I wrote down that it will rotate from 0, which means it will start rotating from 0. Then I rotated both of the ::after/::before differently. The background-color:transparent will make the top of the arrow (or cross if it isn’t transparent) hidden or aka transparent.

I had the same thought with the other class I will add on JavaScript.

Next, with an addeventlistener I will add and toggle the class active on the dropdown menu.

Now it is part for the hamburger menu. First I had to animate it to and be displayed only when the screen is small. The second part is to open the menu when clicking on the icon.

The first part was not difficult since I only had to use display:none on the nav items when the screen is small and on the hamburger icon when the screen is big. To animate the hamburger icon I combined both CSS and JavaScript. By adding keyframes and transform translate on the lines, I was able to animate each line of the icon. Next I used the same method as the one I did for the pointer: addeventlistener and toggle the added class (this time its called .clicking).

For the second part I used JavaScript. First I stored two classes inside separate variables. Then I used the addeventlistener which toggles the class display on or off.
